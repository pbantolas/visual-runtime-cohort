macOS Harness & Visual Runtime

Architecture diagram for the graphics engine mentoring cohort

macOS Harness
Swift App Layer
HostApp
SwiftUI @main app entry point.
Owns the app lifecycle.
MetalView
NSViewRepresentable bridging
SwiftUI to a CAMetalLayer.
EngineSession
Orchestrates EngineHost and
schedules per-frame ticks.
C++ Host Layer
EngineHost
Objective-C++ bridge.
Connects Swift to the C++ module.
EngineModule
Manages the dylib lifecycle:
load, bind API, init, tick, reload.
DynamicLibrary
dlopen/dlsym wrapper with
file-watching for hot-reload.
▼
Engine API Boundary
EngineAPI
C struct of function pointers:
init, resize, update, shutdown.
SurfaceDescriptor
macOS Metal layer handle + width/height
passed to the engine on init.
EngineState
Mutable frame counter & elapsed time
the engine updates each tick.
engine_get_api
extern "C" entry point the host
calls to retrieve the API table.
▼
Visual Runtime (reloadable dylib)
Engine Entry Layer
engine.cpp
Static API table + lifecycle impls
that forward to the Renderer.
Renderer Layer
Renderer
Backend-agnostic frontend.
Type-erases RendererBackend via pimpl.
RendererBackend
Backend-specific implementation interface.
Concrete definition lives in each backend partition.
Backend Partitions
Metal
MTL::Device, CommandQueue,
RenderPipelineState, Buffers.
Vulkan
VkDevice, Swapchain, CommandPool,
DescriptorSets, Dynamic Rendering.
